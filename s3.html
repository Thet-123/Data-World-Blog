Developer Softskill

Category : Softskill

Soft skill တွေရေးလက်စနဲ့ ဆက်ရေးဖြစ်ရင်း နောက်ဆုံးပိတ်အနေနဲ့ developer တိုင်း မိမိတို့လုပ်ငန်းခွင်မှာထားရှိသင့်တဲ့ soft skills လေးတွေကို ရေးပေးလိုက်ပါတယ်။

Developer တစ်ယောက်ရယ်လို့ဖြစ်လာပြီဆို technical ပိုင်းကြီးပဲကောင်းနေရုံနဲ့ မလုံလောက်ပါဘူး။ resume ထဲမှာတောင် ကိုယ့်ရဲ့ soft skills တွေကဘာတွေရှိတယ်ဆိုတာထည့်ရေးထားတတ်စမြဲပါပဲ။
ဆိုလိုချင်တာက hard skill ရော soft skill ပါ balance ဖြစ်နေဖို့လိုပါတယ်။ ဒီနေ့မှာတော့ အပေါ်မှာပြောထားတဲ့အတိုင်း ယေဘုယျအားဖြင့် အရေးကြီးတဲ့ soft skill လေးတွေ state လုပ်ပေးလိုက်ပါတယ်။

Communication is the 1st.

Communication လို့ဆိုတာနဲ့ internal coworker အချင်းချင်းရော၊ clients တွေနဲ့ပါ ထိတွေ့ဆက်ဆံနေရတာကို ပြေးမြင်မိမှာပါ။ ဒီနေရာမှာ communication skill ကအရမ်းအရေးပါပါတယ်။ ကျနော်တို့ developer တွေအမြဲလိုလို ကြုံတွေ့နေရတဲ့ အခက်အခဲတစ်ခုရှိပါတယ်၊ အဲ့တာက technical terms & information တွေကို technical မသိတဲ့လူတွေကို ပြန်ရှင်းပြရတာပါ။ အဲ့နေရာတွေမှာ ကိုယ့်ရဲ့ communicative skill ကသာ weak ဖြစ်နေလို့ရှိရင် ကိုယ်လုပ်ထားသမျှသဲထဲရေသွန်သလို ဆုံးရှုံးသွားနိုင်ပါတယ်။

communication skill ကို detail classified လုပ်မယ်ဆို ၃မျိုးရှိပါတယ်။

listening skill

quality ကောင်းတဲ့ product တစ်ခုကို deliver လုပ်ပေးနိုင်ဖို့အတွက်ဆို နားထောင်နိုင်စွမ်းက အရမ်းအရေးကြီးပါတယ်။ကိုယ့်ရဲ့ boss, clients, co-workers, friends ဘယ်သူနဲ့ပဲဖြစ်ဖြစ် conversation တိုင်းမှာ သေသေချာချာနားစွင့်သင့်ပါတယ်။ အဲ့ဒါက ကိုယ့်ရဲ့ problem solving skill ကိုပါမြင့်တတ်စေလာနိုင်ပါတယ်။ စကားကိုနားထောင်နေတဲ့ အချိန်မှာ eye contact ရှိဖို့၊ စကားထောက်ဖို့၊ ဖြတ်မပြောတတ်ဖို့တွေကိုလည်း ဂရုပြုဖို့လိုပါတယ်။

Verbal skill

ဒီ skill ကတော့ client တွေနဲ့ စကားပြောမှမဟုတ်ပါဘူး၊ ကိုယ့်ရဲ့ အလုပ်ခွင်က လူတွေနဲ့ဆက်ဆံတဲ့ နေရာမှာလဲ လိုအပ်ပါတယ်။ စကားပြောကောင်းပြီး နေတတ်ထိုင်တတ်တဲ့ လူဆို လူချစ်လူခင်ကအစ ပေါပါတယ်။ စကားကောင်းပြောဖို့လိုအပ်တဲ့ presentation , meeting etc လုပ်တဲ့နေရာမှာဆို confidence ရှိရှိ စကားပြောဖို့လိုပါတယ်။ Verbal skill ကောင်းတဲ့ လူတစ်ယောက်က အဲ့အလုပ်ကထွက်သွားရင်တောင် သူစကားပြောကောင်းတယ်ဆိုတဲ့နာမည်က ကျန်ခဲ့မှာပါ။

Nonverbal skill

စကားပြောဆိုတဲ့နေရာမှာ ကိုယ်နေအမူအရာ၊ မျက်နှာထား နဲ့ တစ်ခြားအရာတွေကိုလည်း ဂရုပြုဖို့လိုပါတယ်။

Collaborative skill (Working together)

Collaborative skill ဖြစ်တဲ့ team work skill ကလည်း မရှိမဖြစ်လိုအပ်ပါတယ်၊တစ်ယောက်အပေါ်တစ်ယောက်မှီခိုနေတဲ့ project တွေလုပ်ရတယ်ဆို ပိုလို့တောင်လိုအပ်ပါသေးတယ်။ project တစ်ခုမှာ team members တွေက တစ်ယောက်နဲ့တစ်ယောက် အခေါ်အပြောမရှိဘူး၊ set လုပ်ထားတဲ့ goals & guide lines တွေမရှိဘူး၊ တစ်ယောက်အမြင်ကိုတစ်ယောက်ကလည်း မလေးစားဘူးဆို အဲ့ဒီ project ကတော့ fail ဖြစ်ဖို့ နီးနေပါပြီ။ ကိုယ့်ရဲ့ team members တွေနဲ့အတူ ညှိနှိုင်းဆွေးနွေးနိုင်အောင်ကြိုးစားပြီး project ရဲ့ goals တွေကို သေချာ define လုပ်ထားသင့်ပါတယ်။ conflicts တွေရှိခဲ့လို့ရှိရင်လည်း ပွင့်ပွင့်လင်းလင်း ပြောပြီးဖြေရှင်းသင့်ပါတယ်။ သည်းခံသင့်တဲ့ နေရာမှာလည်း သည်းခံတတ်ဖို့လိုပါတယ်။ team work အကြောင်းနဲ့ပတ်သတ်ပြီးတော့လည်း ပြီးခဲ့တဲ့ ရက်က article တစ်ခုရေးပေးထားပါတယ်။ ဒီမှသွားကြည့်လို့ရပါတယ်။

Better to have good presentation skills

ကိုယ့်မှာရှိတဲ့ technical skills ကောင်းကောင်းတွေကို proof လုပ်ပြနိုင်ဖို့အတွက် presentation ကောင်းကောင်းလုပ်တတ်ဖို့လဲလိုပါတယ်။ ကိုယ့်ရဲ့ client ကိုပဲဖြစ်ဖြစ် Management team ကိုပဲဖြစ်ဖြစ် ကိုယ့်ရဲ့ work in progress ကို case study ဆွဲတာတို့ documentation လုပ်တာတို့ရှိတဲ့ နေရာမှာ တိတိကျကျ နဲ့ပေါ်လွင်အောင် present လုပ်တတ်ဖို့လဲလိုပါတယ်။ document presentation မဟုတ်ဘဲနဲ့ ကိုယ့်ရဲ့ stage performance ကိုလည်း ကောင်းကောင်းလေ့ကျင့်ထားဖို့လိုပါတယ်၊ ပရိသတ်ကိုဘယ်လိုဆွဲဆောင်မလဲ ၊ ကိုယ်ပြောချင်တဲ့ main topic ကိုထိရောက်အောင်ဘယ်လိုပြောမလဲ စသည်ဖြင့် သေသေချာချာလေ့ကျင့်ထားဖို့လိုပါတယ်။ အပေါ်မှာပြောထားတဲ့ nonverbal communication skill ကလည်း ဒီနေရာမှာအရေးပါတဲ့အရာတစ်ခုပဲဖြစ်ပါတယ်။

Empathize & Positive thinking skills

သည်းခံတတ်ခြင်းက ကိုယ်လိုတဲ့ goals တွေကို မြန်မြန်ဆန်ဆန်ရအောင် ကူညီပေးနိုင်ပါတယ်။ ဒါကြောင့် ဘယ်သူနဲ့ပဲဖြစ်ဖြစ် ပေါင်းသင်းဆက်ဆံတဲ့အခါမှာ ခင်ခင်မင်မင်ဆက်ဆံပါ၊ အကောင်းဘက်ကနေမြင်ပြီး အကြံပေးပါ၊ကိုယ့်ကို အကြံပေးရင်လည်းလက်ခံတတ်ပါစေ။ doubts တွေကို တတ်နိုင်သလောက်လျော့ချပြီး မသိတာ၊ မရှင်းတာရှိရင် ပွင့်ပွင့်လင်းလင်း မေးတတ်ပါစေ၊ ဖြေရှင်းတတ်ပါစေ။ conflicts ရယ်လို့ ဖြစ်လာခဲ့လို့ရှိရင်လည်း handle ကောင်းကောင်းလုပ်နိုင်အောင် ပြင်ဆင်ထားသင့်ပါတယ်။ ဒီလိုတွေလုပ်လိုက်ခြင်းအားဖြင့် ကိုယ့်ရဲ့ လုပ်ငန်းခွင် ပတ်ဝန်းကျင်ကလဲ ပိုပြီးတော့ healthy ဖြစ်လာမှာပါ။

Management (Plan, Prioritize & finally deliver)

Clients တွေနဲ့ meeting တွေအပြီးတိုင်း ရလာတဲ့ tasks တွေကို management ကောင်းကောင်းလုပ်ပြီး control လုပ်တတ်ဖို့လဲလိုပါတယ်။ deadline နဲ့ ချိန်ပြီးတော့ ကိုယ့်အတွက် self due date ပြန်ချတာတို့ deliver လုပ်ရမယ့် task တွေကို plan ပြန်ချပြီးတော့ ဘယ် task ကတော့ဖြင့် ဘယ်လိုအရေးကြီးတယ် စသည်ဖြင့် plan & prioritize ကောင်းကောင်းလုပ်တတ်ဖို့လိုပါတယ်။ project deadline အမှီ smoothly deliver လုပ်နိုင်ဖို့ အတွက်မိမိကိုယ်တိုင်ကလည်း self management ကို ကောင်းကောင်းနိုင်နေဖို့လိုပါတယ်။ project management ဆိုတဲ့ အရာဟာ project manager မှ မဟုတ်ပါဘူး ၊ ကျနော်တို့ devs တွေကိုယ်တိုင်လည်းသိထားသင့်တဲ့ အရာတစ်ခုဖြစ်ပါတယ်။

အိုကေ၊ တော်တော်လည်း စုံနေပြီလို့ထင်ပါတယ်။ ထုံးစံအတိုင်း summarize လုပ်ကြရအောင်။ ဒီ career ကိုရွေးပြီး လျောက်ပြီဆိုရင် technical skill တစ်ခုတည်းနဲ့မလုံလောက်ဘူးဆိုတာကို အားလုံးလည်း သဘောပေါက်လောက်ကြပြီလို့ထင်ပါတယ်။ တစ်ချို့ အခြေအနေတွေမှာ soft skill က ပိုတောင် အရေးပါနေတဲ့ အချိန်တွေရှိပါတယ်။ ဒါကြောင့်မို့ soft skill တွေကိုလည်းထပ်ပေါင်းလေ့ကျင့်ရင်း ဒီထက်ပိုကောင်းတဲ့ working environment တွေနဲ့ career success တွေအများကြီးရကြပါစေဗျာ။
