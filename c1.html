Be fluent in english

Fundamentals First

Focus on understanding rather than rushing . မြန်မြန် မြန်မြန် ပြီးဖို့ဆိုတာထက် တစ်ကယ်နားလည်ပြီး ပြီးသွားဖို့လိုပါတယ်။စစလေ့လာတဲ့အချိန်မှာ အရေးကြီးတဲ့ concepts တွေပါတဲ့အတွက်ကြောင့် ကိုယ်တကယ်နားလည်အောင် သေသေချာချာလေ့လာ လေ့ကျင့်ထားသင့်ပါတယ်။ အခြေခံကျေညက်နေရမယ့်အပိုင်းမှာ ပေါ့လျော့ထားခဲ့လို့ရှိရင် advance ပိုင်းတွေရောက်လာလို့ရှိရင် အခက်အခဲတွေကြုံလာရမှာကျိန်းသေပါတယ်။ အရေးပါတဲ့ fundamental အပိုင်းတွေကို skip လုပ်သွားတာတို့ အပေါ်ယံကြည့်သွားတာတို့ မလုပ်ဖို့အကြံပေးပါတယ်။

Focus

ကိုယ်လုပ်နေတဲ့အရာအပေါ်မှာ အပြည့်အဝစိတ်ပါဝင်စားဖို့လိုပါတယ်။ စိတ်မပါလက်မပါလုပ်နေလို့ရှိရင် တစ်သက်လုံးသင်နေတောင်တတ်မှာမဟုတ်ပါဘူး။ ကိုယ့်ရဲ့အချိန်တွေထဲက Learning လုပ်ဖို့အတွက်ဆိုပြီးသက်သက်ထုတ်ထားပြီးတော့ attention ကောင်းကောင်းနဲ့လုပ်မယ်ဆိုရင် သင်ယူတဲ့ နေရာမှာမြန်လာမယ့်အပြင် ကိုယ်နောက်ထပ်ဖတ်/လေ့လာရမယ့် topic ကိုလည်း interest ဖြစ်သွားတဲ့အတွက် motivated လုပ်ပြီးသားဖြစ်သွားပါလိမ့်မယ်။ အရေးကြီးတဲ့ နောက်တစ်ခု က interest မွေးမြူဖို့ပါ၊ ဆိုလိုချင်တာကကိုယ် learning လုပ်နေတဲ့အရာနဲ့ အမြဲတမ်းလိုလို keep in touch ဖြစ်နေဖို့ပါ၊ ဥပမာ ကျနော်တို့ python လေ့လာနေတယ်ဆိုပါစို့၊ နေ့စဉ်သုံးဖြစ်နေတဲ့ social media မှာ python နဲ့ပတ်သတ်တဲ့ news တွေ communities တွေ follow လုပ်ထားလို့ရတယ်၊ even at least memes တွေပါကြည့်လို့ရတယ်။ သွယ်ဝိုက်သောအားဖြင့် နေ့တိုင်း လိုလို ထိတွေ့မှု ရှိနေတဲ့အတွက်ကြောင့် python ပေါ်မှာပိုပိုပြီးစိတ်ဝင်စားလာစေပါတယ်။ Meme တွေပဲ ကြည့်ခိုင်းတာဟုတ်ဘူးနော် နေ့တိုင်း အချိန်တစ်ခုပေးပြီး Learning လုပ်နေဖို့လဲလိုပါတယ်။

Break it down into manageable chunks

ကိုယ့်ရဲ့ Learning Road Map ကို စကတည်းကကြိုဆွဲထားတာကောင်းပါတယ်၊ ဒီနေ့မှာ ဘယ်အပိုင်းကို ဖြတ်မယ်၊ မနက်ဖြန်မှာဘယ်လိုပြန်လေ့ကျင့်မယ်စသည်ဖြင့် Planning လုပ်ထားမယ်၊ တစ်ရက်တည်းနဲ့ doc အပြီးဖြတ်မယ်ဘာညာထက်စာလို့ရှိရင် ကိုယ်တစ်ကယ်လုပ်နိုင်တဲ့အတိုင်းအတာကို အစတည်းကခန့်မှန်းထားပြီး အပိုင်းတွေစနစ်တကျခွဲထားပြီး စီမံတတ်ဖို့လဲလိုအပ်ပါတယ်။

Final Tips

community ကိုကောင်းကောင်းအသုံးချတတ်ဖို့လိုပါတယ်၊ code problems တွေအတွက် help တောင်းလို့ရတယ်၊ learning resource တွေမေးလို့ရတယ်။ tutorials ကြည့်ပြီး ဖတ်ပြီးလုပ်တယ်ဆိုရင် ကွန့်ကြည့်ပါ။ ဖတ်ပြီး ဘယ်လိုအလုပ်လုပ်တယ်ဆိုတာကိုနားလည်ရုံသာမက ကိုယ့်ဘာသာ ကိုယ့် Idea နဲ့ကွန့်ပြီးရေးကြည့်လို့ရှိရင် ကိုယ့်ရဲ့ coding skill က သိသိသာသာ လျင်လျင် မြန်မြန်တတ်လာမှာပါ။

code debugging လုပ်နေတဲ့ အချိန်မှာ break ယူဖို့မ မေ့ပါနဲ့၊ ကိုယ် ၃၊ ၄ နာရီလောက်မနားတမ်းရှာတာမတွေ့ ပေမယ့် ခဏတစ်ဖြုတ်နားပြီး ပြန်စဉ်းစားလိုက်ရင် နည်းလမ်းတွေပေါ်လာတတ်ပါတယ်။

နောက်ဆုံးအနေနဲ့ အချိန်နဲ့ ကြိုးစားအားထုတ်မှုသာ စနစ်တကျပေါင်းစည်းလိုက်နိုင်မယ်ဆိုရင် ဘယ်အရာမဆို မြန်မြန်ဆန်ဆန် လေ့လာသင်ယူနိုင်မှာပါ။ အားလုံးပဲ quick learner တွေဖြစ်ပါစေလို့ဆုတောင်းပေးလိုက်ပါတယ်ဗျာ။
